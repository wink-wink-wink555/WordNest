## 💡 Pourquoi ce projet

En tant que personne ordinaire torturée par le vocabulaire anglais (et qui l'est toujours en fait...), j'ai essayé toutes les applications d'apprentissage de vocabulaire disponibles sur le marché... (´･ω･`)

**Mais la réalité est cruelle :**
- 🤢 **UI anti-intuitive** : soit trop tape-à-l'œil sans focal point, soit trop moche pour être regardée 😟
- 😵 **Répétition des mots connus** : réviser des mots déjà maîtrisés, perdre du temps et saper la confiance 🥺
- 📚 **Océan de vocabulaire déroutant** : des dizaines de milliers de mots qui donnent le vertige, ne pas savoir par où commencer 🥹
- ⚡ **Efficacité déplorable** : après des heures d'étude, on réalise qu'on fait du travail répétitif, et les vrais mots nouveaux ne sont que quelques-uns 😭

**Quand j'en ai eu assez, je me suis dit :** puisque les existants ne sont pas bons, autant en créer un moi-même ! (ง •̀_•́)ง

Ainsi est né WordNest~ Un véritable carnet de vocabulaire personnel axé sur **vos mots nouveaux**, pour dire adieu à la répétition inefficace et faire en sorte que chaque minute d'apprentissage compte !

**Développeur** : [@wink-wink-wink555](https://github.com/wink-wink-wink555) | Un étudiant blessé par les applications de vocabulaire (◕‿◕)

## ✨ Fonctionnalités du projet

### 📚 Carnet de vocabulaire personnel
- **Bibliothèque de vocabulaire exclusive** : ne collecte que les mots nouveaux que vous ajoutez vous-même, fini le travail inutile ! (๑•̀ㅂ•́)و✧
- **Système de marquage intelligent** : mettez des étoiles sur les mots clés, attaquez les points difficiles ciblés
- **Personnalisation** : votre bibliothèque de vocabulaire, vous êtes le maître ! Chaque mot est votre "vieil adversaire"

- **Magie du graphe de connaissances** : API DeepSeek pour dessiner des cartes d'associations lexicales, découvrir les liens mystérieux entre les mots ✨

- **Design réactif** : interface fluide et lisse, révisez votre vocabulaire n'importe où, n'importe quand (´∀｀)♡

- **Animations interactives** : transitions fluides, même changer de mot est un plaisir visuel

- **Exécution locale** : votre carnet de vocabulaire ne vous appartient qu'à vous, confidentialité 100% protégée (｡◕‿◕｡)
- **Utilisation hors ligne** : révisez sans connexion, pas de temps perdu dans le métro
- **Export de données** : format CSV pour sauvegarde à tout moment, plus peur de perdre les données

## 🚀 Démarrage rapide

### Prérequis

- Python 3.8+ (supporté par la plupart des ordinateurs modernes~ )
- 2GB d'espace disque disponible (donnez une maison chaleureuse à votre vocabulaire)
- API DeepSeek pour les fonctions magiques IA ✨

### Étapes d'installation

1. **Adoptez le projet chez vous** (づ｡◕‿‿◕｡)づ
```bash
git clone https://github.com/wink-wink-wink555/WordNest.git
cd WordNest
```

2. **Donnez une petite chambre indépendante au projet**
```bash
# Utilisateurs Windows regardez ici ↓
python -m venv venv
venv\Scripts\activate

# Utilisateurs macOS/Linux regardez ici ↓
python3 -m venv venv
source venv/bin/activate
```

3. **Installez les outils nécessaires**
```bash
pip install -r requirements.txt
```

4. **Configurez la clé API** (configurez si vous voulez les fonctions IA~)

  Le projet utilise `config.py` pour la gestion de configuration. Éditez le fichier `config.py`, remplacez les valeurs par défaut par votre configuration :
  
```python
DEEPSEEK_API_KEY = 'votre_clé'
```

5. **Lancez votre petit nid de vocabulaire** ✨
```bash
python app.py
```

6. **Commencez votre voyage dans le carnet de vocabulaire**
   
   Ouvrez votre navigateur et accédez à : http://127.0.0.1:5000
   
   Félicitations ! Votre carnet de vocabulaire personnel est en ligne ! (ﾉ◕ヮ◕)ﾉ*:･ﾟ✧

## 🏗️ Architecture du projet

Ce projet adopte une **conception d'architecture moderne en couches**, structure de code claire, facile à maintenir et étendre :

```
WordNest/
├── 📄 app.py                    # Point d'entrée principal Flask (pattern usine d'applications)
├── ⚙️ config.py                 # Gestion de la configuration de l'application
├── 🗃️ models.py                 # Définition des modèles de base de données
│
├── 📁 routes/                   # 🎯 Couche de routage (BluePrints)
│   ├── __init__.py              # Initialisation du module de routage
│   ├── word_routes.py           # CRUD vocabulaire + fonction d'auto-test
│   ├── graph_routes.py          # Affichage du graphe de connaissances
│   └── api_routes.py            # Interfaces API RESTful
│
├── 📁 services/                 # 💼 Couche de service (logique métier)
│   ├── __init__.py              # Initialisation du module de service
│   ├── word_service.py          # Logique métier vocabulaire
│   ├── llm_service.py           # Service IA LLM
│   └── graph_service.py         # Service graphe de connaissances
│
├── 📁 utils/                    # 🛠️ Modules utilitaires
│   ├── __init__.py              # Initialisation du module utilitaire
│   ├── constants.py             # Définition des constantes
│   └── settings.py              # Gestion des fichiers de paramètres
│
├── 📁 static/                   # 🎨 Ressources statiques
│   ├── style.css                # Fichier de style
│   ├── script.js                # Script principal
│   └── js/
│       └── vis-network.min.js   # Bibliothèque de visualisation de graphes
│
├── 📁 templates/                # 🖼️ Templates HTML
│   ├── index.html               # Page principale (auto-test vocabulaire)
│   ├── word_list.html           # Liste de vocabulaire
│   ├── add_word.html            # Ajouter un mot
│   ├── add_definition.html      # Ajouter une définition
│   ├── edit_word.html           # Éditer un mot
│   └── knowledge_graph.html     # Graphe de connaissances
│
├── 📁 instance/                 # 🗄️ Dossier de base de données
│   └── words.db                 # Base de données SQLite
│
├── ⚙️ settings.json             # Paramètres de l'application (généré localement)
├── 📋 settings.example.json     # Template de paramètres
├── 🚫 .gitignore                # Règles d'ignorance Git
├── 📋 requirements.txt          # Dépendances Python
└── 📖 README.md                 # Documentation du projet
```

### 🎯 Points forts de l'architecture

**Pattern d'architecture en trois couches - Séparation des préoccupations**

1. **Couche de routage (Routes)**
   - Gère les requêtes et réponses HTTP
   - Validation des paramètres et gestion des erreurs
   - Utilisation de la gestion modulaire Flask Blueprint

2. **Couche de service (Services)**
   - Encapsule la logique métier centrale
   - Modules de fonctionnalités réutilisables
   - Découplage de la couche de routage, testabilité améliorée

3. **Couche de données (Models)**
   - Modèles de données SQLAlchemy ORM
   - Encapsulation des opérations de base de données
   - Support pour la commutation entre plusieurs bases de données

**Avantages de la conception** ✨
- 📦 **Modulaire** : responsabilités claires par couche, facile à comprendre
- 🔧 **Maintenable** : organisation de code raisonnable, faible impact des modifications
- 🧪 **Testable** : logique métier indépendante, facile à tester unitairement
- 🚀 **Extensible** : ajouter de nouvelles fonctionnalités nécessite seulement d'étendre la couche correspondante

## 🔧 Configuration de la magie IA

Vous voulez déverrouiller les superpouvoirs de l'IA ? Suivez les étapes de configuration~ ✨

### 🧠 Paramétrage de l'assistant IA local (Ollama + Qwen)

Faites vivre l'IA dans votre ordinateur, générez des exemples exclusifs et des notes d'apprentissage :

1. **Installer Ollama** (foyer de l'IA locale)
```bash
# Visitez https://ollama.com/ pour télécharger le paquet d'installation
# Comme installer un logiciel ordinaire, cliquez simplement (◡‿◡)
   
# Après installation, téléchargez le modèle Qwen super intelligent
ollama pull qwen2.5:3b
```

2. **Configurer l'API DeepSeek** (magicien du graphe de connaissances)

Éditez le fichier `config.py`, remplacez les valeurs par défaut par votre configuration~

```python
DEEPSEEK_API_KEY = 'votre_clé'
```

### Configuration de la base de données

Par défaut utilise SQLite, supporte d'autres bases de données :

```bash
# Exemple PostgreSQL
DATABASE_URI=postgresql://username:password@localhost/wordnest

# Exemple MySQL  
DATABASE_URI=mysql://username:password@localhost/wordnest
```

### Configuration de déploiement

Déploiement en environnement de production avec Gunicorn :

```bash
# Installer gunicorn (déjà inclus dans requirements.txt)
pip install gunicorn

# Lancer le serveur de production
gunicorn -w 4 -b 0.0.0.0:8000 app:app
```

### 🏗️ Guide de développement d'architecture

Si vous souhaitez ajouter de nouvelles fonctionnalités au projet, veuillez suivre les principes d'architecture en couches suivants :

**1. Ajouter une nouvelle fonctionnalité métier**
```
Étape 1 : Créer ou étendre la classe de service dans services/
Étape 2 : Ajouter le traitement de routage correspondant dans routes/
Étape 3 : Si nouvelle page requise, ajouter le template dans templates/
Étape 4 : Mettre à jour config.py (si nouveaux éléments de configuration)
```

**2. Principes d'organisation du code**
- ✅ **Couche de routage** : gère uniquement les requêtes/réponses HTTP, appelle la couche de service
- ✅ **Couche de service** : contient toute la logique métier, réutilisable
- ✅ **Couche de données** : responsable uniquement des modèles de données et des opérations de base de données
- ❌ **À éviter** : manipulation directe de la base de données dans le routage
- ❌ **À éviter** : traitement des objets de requête HTTP dans la couche de service

**3. Exemple : ajouter une fonctionnalité de favoris**
```python
# Étape 1: services/word_service.py
class WordService:
    @staticmethod
    def toggle_favorite(word_str):
        # Logique métier
        pass

# Étape 2: routes/word_routes.py
@word_bp.route('/favorite/<word>', methods=['POST'])
def favorite_word(word):
    if WordService.toggle_favorite(word):
        return jsonify({'success': True})
    return jsonify({'error': 'Échec de l'opération'}), 400
```
